# qing_southernstudy_events.txt — Southern Study (南書房) panel-lever trampolines — #336 / [#443 boot-crash fix].
#
# The Southern Study panel (QING_southernstudy_panel.txt) used to call its roster/draw/recruit/
# elevate effects DIRECTLY in the scripted_gui button effects. A scripted_gui compile-inlines its
# button's ENTIRE named-effect call-chain at PARSE time (no runtime guard), and every one of those
# chains reaches QING_southernstudy_recompute_roster — whose body is a DEPTH-2 ITERATOR NEST: an
# `every_character { ... }` corps rebuild AND a sorting `ordered_in_list { order_by =
# combined_stats_council_svalue ... }` chief re-derivation (the draw/recruit chains additionally
# reach create_character). The two Study panels (#336 Southern, #337 Upper) are the ONLY mod panels
# at iterator-nest-depth 2, and they were the two group-B boot crashes (EXCEPTION_ACCESS_VIOLATION,
# zero PostValidate) isolated by panel bisection 2026-07-14.
#
# FIX: each effect-bearing button now `trigger_event`s the matching hidden event below. trigger_event
# is a RUNTIME reference, NEVER compile-inlined, so the depth-2 iterator nest lives in an event unit
# the parser does not expand. Behaviour is identical to the old inline effects. (Proven idiom: the
# sibling censorate/harem/justice/princes/personnel trampolines; upstream TI found_city_button ->
# state_improvement.6, Invictus summon_curiate_assembly -> assembly.1.) The indicator + close buttons
# have no effect chain and were left inline.
#
# Namespace = qing_southernstudy
#   .1  open-panel roster refresh   (QING_southernstudy_recompute_roster)
#   .2  DRAW A SCHOLAR (掄才入直)     (QING_southernstudy_draw_scholar)
#   .3  RECRUIT A WILDCARD (高士奇)   (QING_southernstudy_recruit_wildcard)
#   .4  ELEVATE THE CHIEF (擢用)      (QING_southernstudy_promote_chief)
########################################################################
