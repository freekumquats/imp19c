# qing_upperstudy_events.txt — Upper Study (上書房) panel-lever trampolines — #337 / [#443 boot-crash fix].
#
# The Upper Study panel (QING_upperstudy_panel.txt) used to call its roster/choose/recruit/intensive/
# elevate effects DIRECTLY in the scripted_gui button effects. A scripted_gui compile-inlines its
# button's ENTIRE named-effect call-chain at PARSE time (no runtime guard), and every one of those
# chains reaches QING_upperstudy_recompute_roster — whose body is a DEPTH-2 ITERATOR NEST: an
# `every_character { ... }` corps rebuild AND a sorting `ordered_in_list { order_by =
# combined_stats_council_svalue ... }` chief re-derivation (the choose/recruit chains additionally
# reach create_character). The two Study panels (#336 Southern, #337 Upper) are the ONLY mod panels
# at iterator-nest-depth 2, and they were the two group-B boot crashes (EXCEPTION_ACCESS_VIOLATION,
# zero PostValidate) isolated by panel bisection 2026-07-14.
#
# FIX: each effect-bearing button now `trigger_event`s the matching hidden event below. trigger_event
# is a RUNTIME reference, NEVER compile-inlined, so the depth-2 iterator nest lives in an event unit
# the parser does not expand. Behaviour is identical to the old inline effects. (Proven idiom: the
# sibling censorate/harem/justice/princes/personnel trampolines.) The indicator + close buttons have
# no effect chain and were left inline.
#
# Namespace = qing_upperstudy
#   .1  open-panel roster refresh        (QING_upperstudy_recompute_roster)
#   .2  CHOOSE A MASTER (擇師)             (QING_upperstudy_draw_tutor)
#   .3  RECRUIT A RENOWNED MASTER         (QING_upperstudy_recruit_wildcard)
#   .4  INTENSIVE SCHOOLING (勤學不輟)     (QING_upperstudy_intensive_schooling)
#   .5  ELEVATE THE GRAND TUTOR (擢用)     (QING_upperstudy_promote_chief)
########################################################################
