## #1 PICK: Full Imperial Examination System (科舉) — CANDIDATE 1
**Why top-pick:** (1) Existing stub (`qing_exam_ladder` 1-var abstraction) makes this ADDITIVE not disruptive—new system sits BESIDE old, then old is deprecated. (2) CORE mechanic—exam IS Qing governance, not a side-feature. (3) Clean decomposition into parallel phases. (4) Low engine-risk (all idioms proven except char-list-size, which oracle can verify in <30min).

**Decomposition (4-phase workflow, ~20h total):**

### PHASE 1: RESEARCH + DESIGN (2h, 1 agent foreground)
Digest Elman "Cultural History of Civil Examinations" + Miyazaki "China's Examination Hell" + Chaffee "Thorny Gates of Learning". Lock design doc `research/1763_EXAM_SYSTEM_design.md` with: (a) triennial-cycle calendar (provincial 鄉試 autumn Year-1, metropolitan 會試 spring Year-2, palace 殿試 spring Year-2, next cycle +3yr), (b) pass-rate formula (base 5% provincial→300 juren/cycle, 1% metropolitan→100 jinshi/cycle, modified by qing_corruption_level × -0.5%, qing_bureau_integrity × +0.3%), (c) hire-pool integration (graduates enter `qing_scholar_hiring_pool` character-list, `se_QING_PERSONNEL.txt` office-appointment reads list-first before generic-create), (d) crisis-triggers (corruption≥70 → exam-bribery-scandal, Taiping-rebellion → exam-suspension 1850-64, reform-pressure≥80 → abolition-debate), (e) 1763-vs-1815 baselines (1763 pass-rate high/corruption-low, 1815 pass-rate eroded/捐納 prevalent). Return design doc path + 3-sentence summary.

### PHASE 2: ENGINE BUILD (10h, 2-3 agents parallel after oracle-probe)
FIRST: oracle-probe (30min, 1 agent): spawn 2000 `set_as_minor_character` scholars, check RAM/save-size/turn-time; if perf-collapse, cap hiring-pool at 500 + age-out old scholars. THEN parallel: 

**Agent A (5h):** `se_QING_EXAM.txt` core verbs — `QING_exam_init` (seed cycle-year var + hire-pool list), `QING_exam_trigger_cycle` (check current_date mod 3yr = cycle-year, fire qing_exam.1), `QING_exam_phase_provincial/metropolitan/palace` (roll pass-rate, create_character graduates with randomized-skills + exam-tier-trait, add_to_list hire-pool, log count), `QING_exam_crisis_suspend` (set qing_exam_suspended flag, clear on restoration), `QING_exam_reform_abolish` (remove exam-ladder-modifier, fire capstone event, set qing_exam_abolished flag stopping future cycles).

**Agent B (3h):** Hire-pool integration — extend `se_QING_PERSONNEL.txt`: `QING_personnel_appoint_from_scholars` (if qing_scholar_hiring_pool not-empty: pick best-skill match from list for office, remove_from_list, grant office; else: fallback to existing create_character), hook into all 11 GC office-appointment calls. Extend `on_character_death`: if dead-char in hire-pool → remove_from_list (prevent orphan-refs).

**Agent C (2h):** Integration hooks — extend `se_QING_GOVERNANCE.txt`: replace abstract `qing_exam_ladder_target` calc with concrete `set_variable = { name = qing_exam_graduates_recent value = <hire-pool-size-last-3yr> }`, feed into `qing_bureau_integrity_target`. Extend `se_QING_DECLINE.txt` flavour-roll: add exam-crisis branch (if corruption≥70 + NOT qing_exam_suspended: 15% chance → qing_exam.4 bribery-scandal).

### PHASE 3: EVENT CHAINS + LOCALIZATION (6h, 2 agents parallel)
**Agent D (4h):** `events/imp19c_mod_events/qing_exam_events.txt` — 8 events: qing_exam.1 (provincial-phase offer, desc flavor varies by cycle-year, option "Oversee exams" = player-attends for legitimacy+ vs "Delegate" = auto-resolve), .2 (metropolitan-phase, fewer-graduates, higher-stakes desc), .3 (palace-phase, jinshi audience with emperor, prestige+), .4 (corruption-scandal, choice: punish examiner→integrity+ but graduate-count-hit vs ignore→corruption-continues), .5 (provincial-quota-reform memorial, choice: broaden quotas→more-grads/southern-bias vs tighten→quality/northern-resentment), .6 (捐納-purchase-enable proposal, choice: allow→treasury+200k but integrity-crash vs refuse→legitimacy+), .7 (abolition-debate, reformist memorial "exam is obsolete, adopt Western schools", choice: abolish→reform-pressure-relief but bureau-capacity-crater vs defend-tradition), .8 (talent-shortage-crisis if hire-pool empty + office-vacant, emergency 恩科 special-exam offer).

**Agent E (2h):** `localization/english/qing_exam_l_english.yml` (80+ loc keys: event titles/descs/options, modifier names `qing_exam_cycle_active` / `qing_exam_bribery_rampant` / `qing_exam_abolished`, tooltip `QING_EXAM_HIRE_POOL_TT` showing scholar-count + avg-skill).

### PHASE 4: GUI + ADVERSARIAL REVIEW (4h, 2 agents sequential)
**Agent F (2h):** `common/scripted_guis/QING_exam_panel.txt` — L4 exam-policy GUI (button in Grand Council view), window shows: (a) next-cycle countdown-timer, (b) hire-pool scholar-count + avg-skill bars (martial/finesse/charisma/zeal), (c) policy-toggles (捐納 purchase ON/OFF, provincial-quota slider per region), (d) crisis-status flags (suspended/scandal-active/abolition-debate), (e) manual-trigger button "Convene Special Exam 恩科" (costs 50k, fires immediate cycle if NOT on cooldown). ## #2 PICK: Full Tributary System (朝貢體系) — CANDIDATE 2

(2) Extends existing stub (`se_QING_VASSAL.txt` 4 crisis-events) into ACTIVE ongoing-rhythm, high player-engagement. (3) Clean parallel decomposition (research, engine, events, GUI independent). (4) Medium engine-risk (per-vassal schedulers + scope-save envoy need oracle-verify, but worst-case fallback = single shared scheduler + flag-based envoy-tracking).

**Decomposition (4-phase workflow, ~18h total):**

### PHASE 1: RESEARCH + DESIGN (2h, 1 agent foreground)
Digest Fairbank "Chinese World Order" + Hamashita "China, East Asia and the Global Economy" + Larsen "Tradition, Treaties, and Trade". Lock `research/1763_TRIBUTARY_SYSTEM_design.md` with: (a) vassal roster (Korea 3yr, Vietnam 4yr, Ryukyu 2yr, Burma 10yr, Siam 5yr, Nepal 5yr, plus C.Asian Kokand/Bukhara/Khiva as tribute-only not subject), (b) mission-event structure (per-vassal qing_tribute.1_korea / .1_vietnam variants, shared .2 investiture-request template), (c) prestige-feed formula (each successful mission +3 qing_suzerain_prestige, tribute-lapse -5, breakaway-attempt -15), (d) trade-revenue (per-mission +15-25k treasury scaled by vassal-tier, feeds qing_privy_purse if >=1790 Heshen-era), (e) breakaway-trigger (vassal at qing_suzerain_prestige <40 + rival-GP sphere-influence >50 in vassal-area → 20% annual-chance qing_tribute.3 breakaway-crisis), (f) 1763-baseline (prestige 80, all vassals on-schedule) vs 1815 (prestige 60, Korea/Vietnam still-active, Ryukyu-Japan-tension rising).

## #3 PICK: Grand Canal Grain-Tribute Logistics (漕運) — CANDIDATE 3
**Why #3:** (1) Concrete-over-abstract showcase (turns existing abstract `qing_exam_ladder`-style counter into REAL supply-chain with provincial-quotas + shipment-loss + famine-response). (2) Links shallow food-economy into governance/rebellion (famine → unrest → Taiping-recruitment-pool, canonical historical-causation). (3) Medium complexity (lower than exam/tributary char-heavy systems, but arithmetic-heavy). (4) Low engine-risk (all idioms proven: provincial-iteration, numeric-var-millions safe per prior work, food-sums already in se_DEMAND.txt).

**Decomposition (3-phase workflow, ~16h total):**

### PHASE 1: RESEARCH + DESIGN (1.5h, 1 agent foreground)
Digest Carol Shiue "Transport Costs", Philip Kuhn "Soulstealers", Jane Leonard "Controlling from Afar". Lock `research/1763_CANAL_LOGISTICS_design.md` with: (a) Beijing grain-demand formula (base 3M石 capital-pop + 1M banner-garrisons + 0.5M frontier-army = 4.5M total/yr, 1石≈60kg), (b) Jiangnan tribute-quota (Jiangsu 1.2M, Zhejiang 1M, Anhui 0.8M, Jiangxi 0.5M per province-production-capacity), (c) canal-condition bands (efficient 0-20% loss, degraded 20-40%, failed 40%+ = sea-route-mandatory), (d) shipment-loss formula (base 10% weather/banditry + canal-condition-tier×5% + corruption/20×2%), (e) famine-response (if Beijing-reserve <50% demand: choice redirect-tribute-to-famine-relief vs hoard-for-capital, affects harmony±15 + legitimacy±10), (f) 1763-baseline (canal-condition efficient, loss 8%, reserve 120% demand) vs 1815 (degraded, loss 25%, reserve 80%).

### PHASE 2: ENGINE BUILD (9h, 3 agents parallel)
**Agent M (4h):** `se_QING_CANAL.txt` core — `QING_canal_init` (qing_canal_condition=80, qing_grain_reserve_beijing=5400000), `QING_canal_assess_tribute_demand` (set qing_grain_demand_beijing = 4500000 baseline + army-size×50000 + banner-decay-penalty), `QING_canal_collect_tribute_quota` (iterate 4 Jiangnan provinces: subtract province-var caoyun_quota from GOODS_governorship_grain_produced, accumulate qing_grain_tribute_collected), `QING_canal_shipment_loss_roll` (loss-rate-calc per formula, apply to collected-total → qing_grain_delivered), `QING_canal_deliver_grain` (add delivered to reserve, compare to demand, if shortfall: -banner-morale/-harmony nudges), `QING_canal_famine_redirect` (if famine-flag set: choice subtract-reserve→send-to-famine-province +harmony vs refuse -legitimacy), `QING_canal_sea_route_unlock` (if current_date≥1826 + canal-condition<40: enable sea-route-toggle, changes loss-calc to sea-route-banditry-formula).

**Agent N (3h):** Extend `se_QING_WORKS.txt` — `QING_works_pulse` now calls `QING_canal_quarterly_tick` (runs loss-roll + deliver + update-condition-meter; condition drifts: if dredged-recently +5, else if vacant-Works-office -3). Existing qing_works.2 canal-silt event now hooks `QING_canal_update_condition` (on dredge-option: +20 condition, on cheap-option: +5 condition-but-corruption+3, on abandon: condition-crash-to-20).

**Agent O (2h):** Extend `se_QING_DECLINE.txt` — banner-decay now fed by capital-grain-shortage: if qing_grain_reserve_beijing < qing_grain_demand_beijing×0.7: QING_DECLINE_nudge banner_decay +2 (underfed garrisons mutiny). Famine-trigger: in QING_DECLINE_pulse add province-famine-check (if province food-production <50% pop-demand: set province-flag famine_active, fire qing_canal.5 famine-response with that province-scope).

### PHASE 3: EVENTS + GUI + REVIEW (5.5h, 2 agents sequential)
**Agent P (3h):** `qing_canal_events.txt` (7 events) + GUI. qing_canal.1 (quota-setting annual-offer, slider per province 0.5-1.5M, tooltip shows unrest-risk if >80% province-production), .2 (shipment-loss-crisis "Yellow River flooded 清口, 30% tribute-lost", emergency-dredge vs accept-loss), .3 (boatmen-guild-unrest 漕幫 "grain-boatmen strike for wages", pay-bonus vs suppress), .4 (sea-route-proposal "Daoguang era: bypass canal via coastal-shipping?", unlock-toggle), .5 (famine-response "Henan famine, redirect tribute-grain?", redirect/refuse), .6 (breach-at-crossing), .7 (grain-superintendent-embezzlement).
